# AnkiForge AI — v0.2.2 Review Workflow

从 Markdown 笔记提取知识点 → AI provider 生成候选卡片 → 人工预览/编辑 → 写入 Anki → 自动套用美化模板。

v0.2.2 不新增导入源，重点是把预览表格打磨成更适合日常审核的工作台：设置区默认收起、按 Markdown heading 生成、简化主表格、详情弹窗编辑候选卡。

## v0.2.2 做了什么

- 默认仍使用 `mock`，不需要 API key，不联网。
- 保留 provider 选项：
  - `mock`
  - `deepseek`
  - `openai_compatible`
- `deepseek` 自动填入默认 API Base URL：`https://api.deepseek.com`，推荐模型：`deepseek-chat`。
- `openai_compatible` 可用于未来 OpenRouter、SiliconFlow、OpenAI 或其他兼容 Chat Completions 的 API。
- 真实 API 调用走后台任务，避免 Anki UI 长时间假死。
- AI 必须返回结构化 JSON：`{"cards":[...]}`。
- 本地 validator 校验 JSON、字段、`basic` card 类型、Front/Back 非空。
- Prompt 强化 Anki 化要求：每张卡只考一个原子知识点，Front 是单一长期复习问题，Back 直接回答。
- Extra 会保留模型补充信息，并追加 `Generated by provider / model. Please verify manually.`。
- 如果模型没有给 Extra，也会填入最小 provider/model 元信息。
- Tags 会做轻量规范化：去空、去重、英文小写、空白转 `_`、过滤过长标签。
- AI Provider 设置区默认收起，只显示 provider/model 摘要。
- 选择 Markdown 后会显示 heading selector，可生成全部或当前 heading。
- 主表格只显示 Include、Front、Back 摘要、Tags 和状态，Extra/Source 进入详情弹窗。
- 双击候选卡或点击「编辑详情」可编辑 Front / Back / Extra / Tags / Source。
- 增加全选、全不选、反选、删除选中、清空未选中、清空预览。
- 写入成功或全部重复跳过后，当前候选列表会清空，并提示新增/跳过数量。
- Source 字段仍由本地代码生成，模型不能控制 Source。
- AI 只生成候选卡片；必须人工预览、编辑、勾选，再点击写入。
- 写入层继续保留 Tags 字段同步、Anki tags、Front/Back 校验和重复导入跳过。

## v0.3-v0.5 pipeline

v0.3 建立了独立的 import pipeline 数据边界，从 Markdown source、chunks、
knowledge points、human selection、card candidates、quality gate 一直到
human review。完整 mock orchestrator 离线运行，不依赖 Anki 或网络。

v0.4 是 mock pipeline review bridge：现有窗口增加了独立的只读 pipeline
预览和 KnowledgePoint 选择入口，并能展示 Quality、Review 和 Write eligible
状态。PR5 进一步提供纯 Python write eligibility 判断和只读 write-ready
preview，但不生成 `GeneratedCard`，也不调用 Anki writer。

v0.4 review bridge 仍使用 mock pipeline，不会自动批准 HumanReview，也不会自动
写入 Anki。插件原有的 DeepSeek / OpenAI-compatible provider 和人工确认写入流程
继续独立存在，尚未接入新的 pipeline review path。

v0.5 已为新 pipeline 建立 provider contracts、JSON extraction service、安全
包装、OpenAI-compatible provider/HTTP transport 和显式 wiring factory。这些是
纯 Python 基础组件，尚未连接 UI、Anki config、API key storage、orchestrator 或
Anki write path，也不会自动调用真实 provider。原有 v0.2 provider 流程继续独立
运行；新 provider 路径仍然只能产出经过本地 validator 的 KnowledgePoint。

v0.5 provider 安全契约与离线 smoke 路径见
`docs/pipeline_v0_5_ai_provider_contract.md`。即使未来接入真实 provider，输出也必须
继续经过 Human Selection、Quality Gate、Human Review 和用户确认，不能自动写入
Anki。

v0.5 阶段总结与 release checklist 见 `docs/pipeline_v0_5_summary.md`。开发者如需
手动验证真实 OpenAI-compatible provider，可参考 `docs/dev_real_provider_smoke.md`；
该 harness 是 dev-only 工具，会发送明确提供的文本，不是插件 UI 功能，也不会写入
Anki。v0.5 尚未提供普通用户 provider selection、key storage 或 consent flow。

v0.4 的架构和安全契约见 `docs/pipeline_v0_4_review_bridge.md`，Anki 手动验收
步骤见 `docs/anki_manual_acceptance_v0_4.md`。

## 安装到 Anki 测试

1. 找到 Anki 的插件目录：
   - macOS: `~/Library/Application Support/Anki2/addons21/`
   - Windows: `%APPDATA%\Anki2\addons21\`
   - Linux: `~/.local/share/Anki2/addons21/`
2. 把整个 `ankiforge_ai` 文件夹复制进去。
3. 确认文件夹名保持为 `ankiforge_ai`。
4. 重启 Anki。
5. 顶部菜单 **工具 (Tools) → AnkiForge AI** 打开窗口。

## 测试 mock provider

1. Provider 选择 `mock`。
2. 选择 Markdown 文件。
3. 使用「生成/重新生成全部」或选择一个 heading 后生成当前 chunk。
4. 候选卡片进入预览表后，双击某行编辑详情。
5. 勾选要写入的候选卡，再点击「添加到 Anki」。

mock provider 不需要 API key，不会发网络请求。

## 测试 DeepSeek / OpenAI-compatible provider

1. Provider 选择 `deepseek`。
2. 确认 API Base URL 为 `https://api.deepseek.com`。
3. 确认 Model 为 `deepseek-chat`，也可以手动修改。
4. 填入你自己的 API Key。
5. 选择 Markdown 文件，点击「生成卡片」。
6. 等待后台任务完成，候选卡片会进入预览表。
7. 人工编辑、勾选后，再点击「添加到 Anki」。

如果要测试其他 OpenAI-compatible API，选择 `openai_compatible`，手动填写 API Base URL、Model 和 API Key。仓库里的 `config.json` 必须保持 `api_key` 为空字符串，不要提交真实 key。

如果真实 API 失败，可以把 Provider 切回 `mock` 并保存设置；mock 不需要 API key，也不会联网。

## API Key 安全

- 不要把真实 API key 提交到仓库。
- 仓库里的 `config.json` 必须保持 `"api_key": ""`。
- 如果你在 Anki UI 里保存过真实 key，提交前先运行 `git status` 和 `git diff`，确认没有把本地 key 放进提交。
- 公开 issue、日志或截图时，也不要暴露 API key。

## 配置

`config.json` 默认示例：

```json
{
    "ai_provider": "mock",
    "model": "mock-v0.2",
    "api_base_url": "",
    "api_key": "",
    "max_cards_per_chunk": 3,
    "timeout_seconds": 60,
    "temperature": 0.2,
    "default_deck": "AnkiForge::Inbox",
    "default_note_type": "AnkiForge Basic",
    "obsidian_vault_path": ""
}
```

## 手动验收 v0.2.2

- 打开窗口后确认 AI Provider 设置区默认收起。
- 选择 Markdown 后确认状态显示已解析的 chunk 数量。
- 使用 heading selector 只生成某一个 chunk。
- 双击候选卡，确认可编辑 Front / Back / Extra / Tags / Source。
- 使用全选、全不选、反选、删除选中、清空未选中。
- 写入 Anki 后确认提示新增/跳过重复数量，并且预览表被清空。
- 重新导入同一批卡，确认 duplicate check 仍然跳过。

## 不在 v0.2.2 范围内

- PDF importer
- Obsidian vault 扫描
- cloze note type
- 批量导入
- 为 DeepSeek/OpenRouter/SiliconFlow/OpenAI 分别写重复 provider
- 单张卡重新生成
- 生成历史
- API key keychain 存储
