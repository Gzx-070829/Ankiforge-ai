# Security Policy

## Reporting a security issue

Please report suspected vulnerabilities privately through the repository's
**Security** tab by selecting **Report a vulnerability**. If private
vulnerability reporting is not available, open a minimal issue asking the
maintainer for a private contact channel, without including sensitive details.

Do not publish an API key or other credential in an issue, discussion, pull
request, screenshot, or log. API keys must not appear in issue content or log
output shared for debugging.

AnkiForge AI currently uses the API key only for the active session and does
not save it. If you discover that a secret has been exposed, immediately
rotate or revoke the affected key with its provider before reporting the
incident.
