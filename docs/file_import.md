# AnkiForge AI 文件导入说明

v0.11 在现有学习材料文本框上增加了文件选择和拖放入口。导入是完全本地的，只把提取出的文本放进输入框；不会自动调用 AI，也不会自动写入 Anki。

## 支持状态

| 文件类型 | 当前状态 | 限制 |
| --- | --- | --- |
| `.md` / `.markdown` | 内置支持 | UTF-8 优先，保留原始 Markdown，最大 5 MB |
| `.txt` | 内置支持 | UTF-8 优先，保留原始换行，最大 5 MB |
| `.docx` | 内置文本导入 | 最大 10 MB；只提取段落和简单表格文本 |
| `.pdf` | fallback-only | 当前包未内置 PDF 解析器，会提示复制文本或转换格式 |
| `.doc` | 不支持 | 请先另存为 `.docx` |

DOCX 导入不会保留图片、公式、批注、修订历史或复杂排版。

PDF 入口不会做 OCR，也不承诺扫描版、双栏论文、公式或复杂表格的阅读顺序。当前公开构建没有捆绑 PDF 解析器，因此不会在 README 中宣称已完整支持 PDF 文本提取。

## 文件选择与拖放

可以点击“选择文件”，也可以把本地文件拖入学习材料区域。

学习材料输入框使用虚线边框标出可拖放区域；导入成功、限制提示和导入错误使用不同的状态样式。视觉提示不会改变导入流程，也不会触发任何额外操作。

- 一次拖入多个文件时，只导入第一个，并显示提示。
- 输入框为空时，导入文本直接进入输入框。
- 输入框已有内容时，新内容会以文件名分隔线追加到末尾，原有输入不会被静默覆盖。
- 导入成功后会显示文件名、类型和导入字符数。
- 警告和错误只显示用户可理解的文字，不显示内部 traceback。

## 安全边界

- 文件解析不依赖 Anki collection。
- 文件解析不调用 Provider，也不发送网络请求。
- 导入后仍需由用户主动点击“生成卡片”。
- 写入仍要经过审核、查重和最终确认。
- API key 仍只在当前会话使用，不会保存。

## v0.11 人工验收重点

- 分别通过“选择文件”和拖放导入 Markdown、TXT、DOCX。
- 确认成功提示包含文件名、类型和字符数。
- 拖入 PDF 时确认只出现 fallback 指引，不发生网络请求或 OCR。
- 输入框已有内容时确认新文件追加到末尾，原内容仍然存在。
- 切换中英文后确认导入提示自然且信息一致。
