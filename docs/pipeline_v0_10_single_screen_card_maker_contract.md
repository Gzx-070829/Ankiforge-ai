# v0.10 PR10：单屏 AI 制卡面板契约

PR10 将默认产品界面收敛为一个单屏 AI 制卡工具。普通用户只需理解四个同屏区域：学习材料、AI、生成的卡片、写入 Anki；不再通过向导、演练或开发者状态推进流程。

## 默认界面

- 主窗口标题为 “AnkiForge AI”。
- 副标题为 “把学习材料变成 Anki 卡片”。
- 学习材料支持粘贴、选择 Markdown 文件或加载内置示例。
- AI 区默认只显示 Provider、Model 和本次使用的 API key。
- Base URL 与 Timeout 位于默认折叠的高级设置中。
- 生成的卡片默认保留，可编辑或丢弃；来源默认折叠。
- 写入区集中显示 Deck、Note type、字段映射、重复检查和真实写入按钮。
- 旧流程和开发调试功能保留在默认折叠的“高级 / 调试工具”中。

## 写入边界

- 打开窗口不会调用 AI，也不会自动写入。
- 只有用户点击“生成卡片”才会发起 Provider 请求。
- API key 仅存在于当前控件与单次请求设置中，不进入 config、safe summary、写入命令或错误文案。
- 只有至少一张卡片被保留、目标和字段映射完整、重复检查成功后，写入按钮才可用。
- 可能重复或被丢弃的卡片默认跳过。
- 点击“写入 Anki”后必须通过第二次确认；取消不会调用 writer。
- writer 只能创建新 note，不修改 Deck、Note type 或已有 note/card。
- 成功写入过的同一快照不能再次写入。
- 写入结果按成功、失败和跳过分别记录，不误报成功。

## 会话生命周期

材料、API key、AI 输出、选择、映射、重复检查及写入结果均不写入配置。关闭主窗口时清空控件并关闭内存 session。
