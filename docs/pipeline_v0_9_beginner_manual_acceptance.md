# AnkiForge AI v0.9 新手模式手动验收

## 1. 目的

本手册验证 v0.9 的完整新手流程是否可理解、可导航且没有副作用。

**v0.9 是离线只读新手演练版，不是正式自动写入版。** 验收期间不得进入旧版
工作台的真实风险入口，不得输入凭据，不得启动 Provider、writer 或任何网络工具。

## 3. 前置条件

- [ ] `python -m unittest discover -s tests` 通过。
- [ ] `python -m compileall .` 通过。
- [ ] `git diff --check` 通过。
- [ ] Anki 在任何运行文件同步前已经关闭。
- [ ] 同步范围只包含必要运行文件；tests/docs 未复制到 add-on。
- [ ] add-on `config.json` 未复制、覆盖或修改，前后 SHA-256 相同。
- [ ] `addons21` 下只有一个 `ankiforge_ai`。
- [ ] `ankiforge_ai_backup_20260630_005727` 存在于 `addon_backups`。
- [ ] 使用非敏感测试材料。

PR6 为纯文档 PR 时，不需要重新同步运行文件，也不要求重复执行 PR3–PR5 已完成的
UI 手动验收。本手册保留为最终 v0.9 基线和后续用户测试前的复核入口。

## 4. 主入口与模式隔离

1. 启动 Anki。
2. 从工具菜单打开 AnkiForge AI。
3. 观察默认入口。

验收：

- [ ] 主窗口正常打开且没有异常。
- [ ] “新手模式（推荐）”清晰、突出。
- [ ] “旧版工作台（高级）”与新手入口视觉隔离并带风险提示。
- [ ] 打开主窗口本身不会创建 note、读取 collection 或触发网络。

## 5. Step 1：选择学习材料

1. 打开新手模式。
2. 确认欢迎区、离线只读安全状态条和五步导航可见。
3. 粘贴一段包含多个句子的非敏感学习材料。
4. 点击“继续”。

验收：

- [ ] 材料字符数正确。
- [ ] 页面明确说明内容只保留在当前窗口。
- [ ] 没有凭据、保存、配置或真实执行入口。

## 6. Step 2：查看系统识别了什么

验收：

- [ ] 页面明确说明当前使用离线演练识别，不会联网，也不会调用 AI。
- [ ] 识别结果只来自刚才输入的材料。
- [ ] 空材料不会产生知识点。
- [ ] 页面没有声称调用了真实 AI Provider。

## 7. Step 3：选择要制卡的知识点

1. 只勾选识别结果中的一部分知识点。
2. 点击“继续”。

验收：

- [ ] Step 3 展示的知识点与 Step 2 一致。
- [ ] 可以清楚选择一部分知识点。
- [ ] 选择仅存在于当前 session。

## 8. Step 4：审核候选卡

验收：

- [ ] 候选卡只来自 Step 3 刚选择的知识点。
- [ ] 每张卡显示正面预览、背面预览和来源片段。
- [ ] 每张卡可选择“看起来可以 / 需要修改 / 暂时不要”。
- [ ] 页面明确说明选择只用于本次离线演练，不会写入 Anki。
- [ ] 未完成每张候选卡的选择前，“继续”不会误导用户跳过审核。
- [ ] 没有候选卡时提示返回上一步选择知识点。

返回 Step 3 改变选择后再次进入 Step 4：

- [ ] 旧候选卡和旧审核选择被清除或按新选择重建。
- [ ] 新候选卡仍只来自当前选择。

## 9. Step 5：查看未来仍缺少的条件

验收：

- [ ] 页面先用普通中文说明当前仍只是离线演练。
- [ ] 目标牌组显示“未来需要确认”。
- [ ] 笔记类型显示“未来需要确认”。
- [ ] 字段映射显示“未来需要确认”。
- [ ] 重复检查显示“未来需要确认”。
- [ ] 最终确认显示“未来需要确认”。
- [ ] 页面不暗示审核决定已经构成写入授权。

技术详情：

- [ ] 默认折叠。
- [ ] “查看技术详情”可以展开，“隐藏技术详情”可以收起。
- [ ] 不阅读技术详情也可以完成主流程。
- [ ] 人工审核、未来写入条件、未来写入方式和最终确认说明均明确不代表写入授权。

## 10. 终点与 session 生命周期

1. 点击“继续”到达终点。
2. 确认终点精确显示：**演练完成，尚未写入 Anki**。
3. 关闭新手模式并重新打开。

验收：

- [ ] 上一次材料不存在。
- [ ] 上一次知识点选择不存在。
- [ ] 上一次候选卡和审核选择不存在。
- [ ] 没有本地草稿路径或恢复入口。

## 11. 上游变化清除

重新演练并分别修改上游状态：

- [ ] 修改材料后，识别结果、选择、候选卡、审核和所有下游预览被清除。
- [ ] 修改知识点选择后，候选卡、审核和所有下游预览被清除。
- [ ] 修改候选卡后，审核和所有下游预览被清除。
- [ ] 修改审核决定后，eligibility、write plan 和 final confirmation preview 被清除。
- [ ] `approved`、`eligible`、`ready_preview` 和
      `ready_for_future_confirmation` 均不被解释为写入授权。

## 12. 无副作用复核

结束前确认：

- [ ] 未创建或修改 note。
- [ ] 未修改牌组、笔记类型或字段。
- [ ] 未访问 Anki collection。
- [ ] 未执行 duplicate check。
- [ ] 未运行 Provider。
- [ ] 未联网。
- [ ] 未读取或保存 API Key。
- [ ] 未调用 writer。
- [ ] 未保存材料、选择或审核结果。
- [ ] add-on `config.json` SHA-256 不变。
- [ ] add-on 备份仍保留。

## 13. 已完成验收记录

- [x] PR3 材料页、五步导航、关闭后丢弃：用户确认通过。
- [x] PR4 材料 → 识别 → 选择 → 候选卡上下文贯通：用户确认通过。
- [x] PR5 三态人工审核、五项未来条件、技术详情折叠：用户确认通过。

PR6 只新增封板文档，不更改运行代码，因此无需再次同步 add-on 或重复 UI 验收。

## 14. 失败处理

任一安全项失败时：

1. 立即停止验收；
2. 不尝试启用或修补真实执行路径；
3. 保留仓库、config 和备份现场；
4. 记录精确步骤、界面文案和观察到的副作用；
5. 在独立、明确授权的后续 PR 中处理。
