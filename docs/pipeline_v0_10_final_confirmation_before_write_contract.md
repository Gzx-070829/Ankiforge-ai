# v0.10 PR7 真实写入前最终确认预览契约

## 定位

PR7 在新手模式第五步增加 beginner-only 最终确认预览。它集中展示当前内存会话中的候选卡、人工审核、目标牌组、笔记类型、字段映射和只读重复检查结果。

页面固定说明：

- 当前只是最终确认预览，尚未写入 Anki。
- 真实写入将在下一步版本开放。
- 演练完成，尚未写入 Anki。

本 PR 没有真正写入按钮，也不生成写命令、正式待写入卡片或写入就绪对象。

## 汇总内容

最终确认预览显示：

1. 候选卡总数，以及每张卡的正面、背面和来源。
2. 每张卡的人工审核选择。
3. 每张卡的只读重复检查状态。
4. 当前选择的目标牌组、笔记类型和字段映射。
5. 应跳过或需要注意的卡片提示。

如果缺少候选卡、完整审核、目标牌组、笔记类型、正面/背面字段映射或重复检查，预览会逐项说明缺失条件。即使条件齐全，也只表示信息已汇总，不代表写入授权。

## 状态与生命周期

最终确认预览只存在于当前窗口内存中。以下变化会立即使旧预览失效：

- 候选卡变化；
- 人工审核变化；
- 目标牌组、笔记类型或字段映射变化；
- 开始、重新执行、清除或失败的重复检查；
- 材料、Provider、model、base URL 或本次会话 API key 变化。

关闭窗口后，材料、AI 草稿、审核、目标选择、重复检查和最终确认预览全部丢弃。

## 安全边界

- 允许读取当前内存 session 的预览状态。
- 继续沿用现有显式触发的只读 Anki 结构读取与重复检查。
- 不写 Anki，不创建或修改 note/card。
- 不修改 deck、note type 或 collection。
- 不调用 writer，不执行真实写入。
- 不保存 API key、材料或 AI 输出，不写 config。
- 自动测试只使用内存模型和既有 fake/mock collection，不访问真实 Anki collection，不真实联网。
