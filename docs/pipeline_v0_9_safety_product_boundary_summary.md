# AnkiForge AI v0.9 安全与产品边界摘要

## 1. v0.9 是什么

**v0.9 是离线只读新手演练版，不是正式自动写入版。**

它把 v0.8 的审核与未来条件概念翻译成普通用户可理解的五步流程：

```text
选择学习材料
  → 查看离线 mock 识别结果
  → 选择要制卡的知识点
  → 审核候选卡
  → 查看未来真正写入前还缺少的条件
  → 演练完成，尚未写入 Anki
```

整个流程只存在于一个窗口内的 `BeginnerFlowSession`。关闭窗口即丢弃。

## 2. v0.9 当前能力

- 打开默认推荐的新手模式；
- 粘贴学习材料并查看字符数与预览；
- 用确定性的本地规则生成离线 mock 知识点；
- 选择知识点；
- 生成只读候选卡预览，且保留与已选知识点的来源关系；
- 用“看起来可以 / 需要修改 / 暂时不要”留下本次审核选择；
- 查看目标牌组、笔记类型、字段映射、重复检查和最终确认等未来条件；
- 按需展开普通中文技术说明；
- 在终点明确看到“演练完成，尚未写入 Anki”。

## 3. v0.9 不具备什么

v0.9 新手模式不具备：

- 真实 AI Provider 调用；
- API Key 输入、读取或保存；
- 网络访问；
- 真实 duplicate check；
- Anki collection 访问；
- note、牌组、笔记类型或字段变更；
- Anki 写入；
- 用户材料、选择、候选卡或审核结果持久化；
- 正式 `GeneratedCard` 或 `WriteReadyPreviewItem` 输出；
- writer 对象或 writer 调用；
- 写入授权、最终确认或同意记录。

旧版工作台仍作为高级区域保留，并与新手模式视觉隔离。旧版区域的存在不改变新手
模式边界，也不表示 v0.9 新手流程会调用旧版执行路径。

## 4. 数据生命周期

新手 session 只在内存中保存当前窗口需要的材料、识别预览、选择、候选卡和审核
选择。它没有持久化文件路径、草稿恢复路径、secret store、Provider 配置、collection
句柄、writer 或写入授权字段。

清除规则：

- 修改材料会清除识别结果及所有下游状态；
- 修改知识点选择会清除候选卡及所有下游状态；
- 修改候选卡会清除审核及所有下游状态；
- 修改审核决定会清除未来 eligibility、write plan 和 final confirmation preview；
- 关闭窗口会清除整个 session，并禁止复用。

`repr` 和安全字典只暴露长度、计数与状态，不输出完整材料、卡片内容或审核备注。

## 5. 状态含义边界

以下状态只能描述只读演练，不得被提升解释：

```text
approved != 写入授权
eligible != 写入授权
ready_preview != 可执行的写入计划
ready_for_future_confirmation != 最终确认
```

“看起来可以”只表示测试用户对候选卡预览的本次选择，不授权任何外部动作。
Step 5 的每一项都显示“未来需要确认”，不表示条件已经满足。

## 6. 配置、add-on 与 Git 边界

- add-on `config.json` 不属于新手 session，运行文件同步不得覆盖它；
- 同步前后必须核对 `config.json` SHA-256；
- `addons21` 下只允许一个 `ankiforge_ai` 运行目录；
- 回滚备份保留在 `Anki2/addon_backups`，不得删除；
- tests/docs 不同步到 add-on 运行目录；
- 未经单独授权不 fetch、pull 或 push；
- 本地开发继续保持 private。

## 7. 用户测试边界

v0.9 封板后的下一步是邀请 1–3 位真实用户做流程理解测试。测试要回答：

- 用户是否知道当前不会写入 Anki；
- 用户是否知道自己位于哪一步；
- 用户是否理解候选卡来自刚才选择的知识点；
- 审核页是否像普通用户页面；
- 哪些词引发害怕、困惑或错误预期；
- 用户是否理解关闭后内容会消失。

用户测试不评估真实 AI 质量、duplicate check 或 Anki 写入，因为这些能力不属于
v0.9。

## 8. 私有与未来版本边界

v0.9 不做 public repo 准备，不新增开源治理文件，也不以公开发布语气描述当前状态。
是否开源、何时开源及相应 README、LICENSE、安全政策和贡献流程，留待 v1.0 的独立
决策和授权。

任何真实 Provider、duplicate check、collection 或 writer 能力都必须作为独立后续
工作，具备明确范围、自动测试、手动验收和单独授权；不得从 v0.9 封板推导。

## 9. 封板声明

PR1–PR5 建立了状态模型、入口隔离、材料导览、上下文贯通和人工审核渐进披露。
PR3、PR4、PR5 的 Anki UI 手动验收均由用户确认通过。PR6 仅补齐用户测试准备、
整体验收与安全产品边界文档。

当 PR6 自动验证通过并 merge 到本地 `main` 后，v0.9 封板为：

> **离线只读新手演练版，可用于 1–3 位真实用户的流程理解测试。**

该封板不授权联网、Provider、collection、duplicate check、writer、Anki 写入、公开
发布或 push。
