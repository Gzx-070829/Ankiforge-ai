# AnkiForge AI v0.9 新手用户测试准备清单

## 1. 用途

本清单用于封板 v0.9，并判断它是否可以交给 1–3 位真实用户做流程理解测试。

**v0.9 是离线只读新手演练版，不是正式自动写入版。** 本次封板只证明
新手流程可以被理解和演练，不代表任何真实执行能力已经启用，也不构成公开发布或
开源准备。

## 2. 当前可测试能力

测试用户应当能够完成以下流程：

- [ ] 打开“新手模式（推荐）”。
- [ ] 粘贴一段学习材料。
- [ ] 查看仅基于当前材料产生的离线 mock 识别结果。
- [ ] 选择想继续查看的知识点。
- [ ] 看到只来自已选知识点的候选卡。
- [ ] 用“看起来可以 / 需要修改 / 暂时不要”审核候选卡。
- [ ] 查看未来真正写入前还缺少的目标牌组、笔记类型、字段映射、重复检查和最终确认。
- [ ] 按需展开或隐藏技术详情；技术详情不是完成演练的必读项。
- [ ] 在终点看到精确文案：**演练完成，尚未写入 Anki**。
- [ ] 关闭窗口后重新打开，前一次材料、选择、候选卡和审核结果均不存在。

## 3. 当前明确不具备的能力

测试前必须向参与者说明，v0.9 不具备：

- [ ] 真实 AI Provider 调用；
- [ ] API Key 输入、读取或保存；
- [ ] 联网；
- [ ] 真实 duplicate check；
- [ ] 访问 Anki collection；
- [ ] 写入 Anki 或创建 note；
- [ ] 保存用户材料、知识点选择或审核结果；
- [ ] 生成正式 `GeneratedCard` 或 `WriteReadyPreviewItem`；
- [ ] writer 调用；
- [ ] 任何由审核状态推导出的写入授权。

## 4. 参与者与测试范围

- 建议参与者：1–3 位没有参与开发、对 Anki 熟悉程度不同的真实用户。
- 单次时长：15–25 分钟。
- 测试目标：理解、导航、词汇和风险认知，而不是卡片质量或自动化能力。
- 材料要求：使用非敏感、可公开或专门准备的短学习材料。
- 观察方式：让参与者先独立操作，观察者只在卡住时记录，不代替点击。
- 退出条件：参与者误以为发生真实写入、看到危险入口、发现内容被保留，或观察到任何
  collection、Provider、网络或 note 副作用时立即停止。

## 5. 面向测试用户的问题

操作过程中和结束后记录原话：

1. 你是否知道当前不会写入 Anki？是在哪个位置知道的？
2. 你是否始终知道自己位于五步流程的哪一步？
3. 你是否理解 Step 2 的结果来自当前材料的离线演练规则？
4. 你是否理解 Step 4 的候选卡来自刚才选择的知识点？
5. “看起来可以 / 需要修改 / 暂时不要”是否足够清楚？
6. 审核页是否像普通人能理解的页面？哪里仍像开发工具？
7. Step 5 是否让你理解未来还缺哪些条件？
8. 你是否误以为某个状态代表已经获得写入权限？
9. 哪些词让你害怕、困惑或产生错误预期？
10. “查看技术详情”是否容易发现？你是否认为它是必读项？
11. 关闭后内容消失是否符合预期？关闭前是否已经知道会丢弃？
12. 如果只能修改一个地方，你最希望改哪里？
